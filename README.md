# DinaStyleKnits — сайт для Digital-товарів

Багатосторінковий сайт-каталог в'язальних PDF-патернів. Бекенд — FastAPI +
SQLAlchemy (Варіант B з обговорення), фронтенд — чистий HTML/CSS/JS
(без фреймворків, щоб не ускладнювати простий каталог).

## Структура проєкту

```
dinastyleknits/
├── backend/               # FastAPI застосунок
│   ├── app/
│   │   ├── main.py         # точка входу, підключення роутерів
│   │   ├── config.py       # налаштування з .env
│   │   ├── database.py     # SQLAlchemy engine/session
│   │   ├── models.py       # Pattern, Subscriber, AdminUser, Post(заготовка)
│   │   ├── schemas.py       # Pydantic-схеми
│   │   ├── auth.py         # JWT-авторизація адмінки
│   │   ├── migrations.py   # легкі авто-міграції (як у StoryLore)
│   │   ├── seed_data.py    # наповнення БД реальними даними з Etsy
│   │   └── routers/        # patterns.py, subscribers.py, auth_router.py
│   ├── static/images/      # фото 13 патернів (їх бачить бекенд)
│   ├── requirements.txt
│   └── .env.example
└── frontend/               # статичний сайт
    ├── index.html          # Home
    ├── patterns.html        # повний каталог (усі патерни)
    ├── pattern-detail.html # повна сторінка товару (?slug=...)
    ├── new.html             # New releases
    ├── blog.html            # заглушка (YouTube-відео додасться пізніше)
    ├── about.html           # заглушка (текст уточнюється замовницею)
    ├── css/style.css        # дизайн-система (кольори/типографіка/адаптив)
    ├── js/                  # config, api, layout, рендеринг карток і сторінок
    └── static/images/       # тільки емблема (брендинг, не змінюється з адмінки)
```

## Запуск бекенду (локально)

```bash
cd backend
python3 -m venv .venv && source .venv/bin/activate   # опційно, але рекомендовано
pip install -r requirements.txt

cp .env.example .env
# відкрити .env і задати ADMIN_EMAIL / ADMIN_PASSWORD / JWT_SECRET

python3 -m app.seed_data      # створює БД, наповнює 13 патернами + адмін-акаунт
python3 -m uvicorn app.main:app --reload --port 8000
```

Після старту:
- API: http://127.0.0.1:8000
- Автоматична документація (Swagger): **http://127.0.0.1:8000/docs** — звідси
  можна одразу тестувати всі ендпоінти вручну (в т.ч. адмінські, увійшовши
  через `/api/auth/login` і вставивши токен у кнопку Authorize)

## Запуск фронтенду (локально)

Фронтенд — статичні файли, тож підійде будь-який простий сервер:

```bash
cd frontend
python3 -m http.server 3000
```

Відкрити http://127.0.0.1:3000/index.html

⚠️ Перед реальним деплоєм змінити `API_BASE_URL` у `frontend/js/config.js`
з `http://127.0.0.1:8000` на реальний URL бекенду (Render).

## Вхід в адмінку (поки без UI — через API/Swagger)

- Email/пароль — ті, що вказані в `backend/.env`
- Адмінського UI (форми додавання/редагування патернів) ще немає — це
  наступний крок розробки. Наразі керувати патернами можна напряму через
  `/docs` (Swagger UI) — POST/PUT/DELETE `/api/patterns` вимагають токен
  з `/api/auth/login`.

## ⚠️ Що обов'язково перевірити перед показом замовниці

1. **Відповідність фото ↔ назви товару.** Прив'язка `image_filename` у
   `seed_data.py` зроблена *best-effort* (за візуальною схожістю), бо Etsy
   не віддає прямих посилань на мініатюри при звичайному запиті сторінки.
   Пройдіться по всіх 13 патернах на `/patterns.html` і звірте з замовницею,
   що кожне фото відповідає правильній назві.
2. **Ціни та посилання на Etsy** — вже реальні дані з магазину, актуальні
   на момент розробки (перевірити, чи не змінились).
3. **Google Fonts** — у пісочниці розробки домен fonts.googleapis.com
   заблокований мережевим проксі, тому шрифт Inter може не підвантажитись
   і випасти на системний sans-serif. На реальному хостингу (Vercel/Render)
   це не проблема — Google Fonts буде доступний як завжди.
